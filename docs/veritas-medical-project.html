Veritas Medical Systems

Knowledge-Grounded Medical AI

Building a medical question-answering system that combines the reasoning capabilities of Large Language Models with the structured, verified knowledge from medical Knowledge Graphs to improve accuracy on complex medical questions and eliminate hallucinations through novel embedding alignment techniques.

Project Overview

Veritas Medical Systems represents a breakthrough in medical AI, addressing the critical challenge of LLM hallucinations in healthcare contexts. By grounding language model responses in verified medical knowledge through innovative embedding alignment techniques, the system achieves both semantic understanding and structural validity, resulting in 5-10% accuracy improvements over standard LLMs on medical QA tasks.

Core Innovation

The Critical Problem: LLMs often generate plausible-sounding but incorrect medical information, which can have serious consequences in healthcare settings.

Our Solution: KGAREVION (Knowledge Graph Augmented Revision) grounds LLM responses in verified medical knowledge through a novel embedding alignment technique that enables the LLM to understand both the semantic meaning AND structural relationships in the Knowledge Graph, providing dual verification for medical facts.

Key Technical Components

Embedding Alignment Module

Bridges KG structural embeddings (TransE) with LLM token embeddings, enabling semantic-structural understanding

Prefix Token System

Injects KG knowledge as special tokens prepended to LLM input for context-aware generation

Description Dictionary

Maps 18 medical relation types to natural language descriptions for improved comprehension

Fine-tuning Pipeline

Trains LLM on KG completion tasks to understand structural patterns in medical knowledge

Dual Verification

Validates both semantic validity and structural existence in knowledge graph

Iterative Refinement

Progressive correction mechanism for complex medical reasoning chains

Full-Stack Implementation

  • Backend Architecture: FastAPI service with WebSocket support for real-time processing, PostgreSQL for audit logs, Neo4j for Knowledge Graph storage, Redis for response caching
  • Frontend Interface: React application with real-time processing visualization, interactive triplet graph display, and confidence scoring visualization
  • Deployment Infrastructure: Docker Compose configuration for local development, Kubernetes manifests with GPU support for production, Auto-scaling based on request load
  • Model Serving: TorchServe for LLM inference, Custom embedding server for KG alignments, Batch processing for efficiency
  • Monitoring & Logging: Prometheus metrics for performance tracking, Grafana dashboards for system health, Structured logging for debugging

# Dual Verification Process
def verify_medical_fact(triplet, kg_embeddings, llm_embeddings):
    """
    Verifies medical facts through dual validation
    """
    # Semantic Validity Check
    semantic_score = cosine_similarity(
        llm_embeddings[triplet],
        medical_context_embeddings
    )
    
    # Structural Validity Check  
    structural_score = kg_validator.check_existence(
        subject=triplet.subject,
        relation=triplet.relation,
        object=triplet.object
    )
    
    # Combined verification
    is_valid = (semantic_score > 0.85) and (structural_score > 0.90)
    confidence = (semantic_score + structural_score) / 2
    
    return is_valid, confidence
